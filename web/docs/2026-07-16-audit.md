## 项目概览

| 维度 | 评估 |
|------|------|
| 规模 | ~15 个 PHP 文件，~2500 行代码 |

#### [x] 1. SQL 注入 — `check_email.php:updatedb()`

[check_email.php:37-39](../src/api/check_email.php#L37) 的 `updatedb()` 使用字符串拼接构造 SQL：

```php
$select = "SELECT * FROM emails WHERE email = '$email'";
$update = "UPDATE emails SET vericode = '$vericode',expire='$expire' WHERE email = '$email'";
$insert = "INSERT INTO emails (email, vericode, expire) VALUES ('$email', '$vericode', '$expire')";
```

虽然上游有 `FILTER_VALIDATE_EMAIL`，但不应依赖过滤器作为唯一防线。

**已修复**：改为 PDO prepared statement。

#### [x] 2. Open Redirect

[login.php:33-35](../src/api/login.php#L33) 和 [logout.php:9](../src/api/logout.php#L9) 直接使用 `$_GET['return_url']` 做 `Location` 重定向，无任何校验。攻击者可构造钓鱼 URL。

**已修复**：限制为相对路径（以 `/` 开头），拒绝外部 URL。

#### [x] 3. SQL 语法错误

[map_request.php:263](../src/api/map_request.php#L263) 普通用户查询"我的申请"的 SQL 有两个拼写错误：

```sql
-- craeted_at → created_at, updated_atmr → updated_at, mr.
SELECT mr.id,mr.steam_id,mr.title,mr.status,mr.craeted_at,mr.updated_atmr.explaination,...
```

**已修复**：`mr.created_at, mr.updated_at, mr.explaination`。

#### [x] 4. 无 CSRF 保护

所有修改操作（登录、注册、删除评论、批准/删除申请、地图管理）都没有 CSRF Token 验证。

**已修复**：在 `auth.php` 中新增 `csrf_token()` + `csrf_hidden_field()` + `verify_csrf()` 函数；所有 POST 端点加入校验；`navbar.php` 的 `<head>` 输出 `<meta name="csrf-token">` 供 JS 读取；`navbar.js` 全局拦截 `fetch` 自动为同源 POST 请求注入 `X-CSRF-Token` header；登录/注册表单嵌入隐藏字段。

#### [x] 5. 本地 IP 认证绕过

[auth.php:8-10](../src/lib/auth.php#L8) 对 127.0.0.1/localhost/::1 完全跳过认证，`check_admin()` 也返回 true。如果存在 SSRF 或 nginx 配置不当，攻击者可获得管理员权限。

**已修复**：移除 IP 白名单，内部调用统一使用 SIDECAR_TOKEN。

#### [x] 6. 验证码无频率限制

`rate_limit()` 函数已存在于 `auth.php`，但 `check_email.php` 未调用。

**已修复**：在 `check_email.php` 中 CSRF 验证后加入 `rate_limit(1, 60)`，限制每 60 秒最多 1 次验证码请求。

#### [x] 7. 会话固定攻击

`login.php:23-27` 中 `session_destroy()` + `session_start()` 但未调用 `session_regenerate_id(true)`。

**已修复**：登录成功时用 `session_regenerate_id(true)` 替代 `session_destroy()+session_start()`，防止会话固定攻击。

#### [x] 8. 硬编码 IP/域名

| 位置 | 硬编码值 |
|------|---------|
| `index.php:127`, `navbar.php:122` | `82.156.112.164:27015` |
| `email.php:159` | `tunarund@tunarund.top` |
| `email.php:163` | `yaokun-handsome@qq.com` |

README TODO 已提到需通过 .env 适配，但未执行。

**已修复**：`config.php` 新增 `SERVER_CONNECT`、`BRAND_EMAIL`、`BRAND_DOMAIN`、`BRAND_REPLY_EMAIL`、`BRAND_COMPANY`、`BRAND_SITE` 常量；`index.php` 和 `navbar.php` 引用 `SERVER_CONNECT`；`email.php` 引用 `BRAND_*` 常量。

#### [x] 9. SES 密钥裸常量

`email.php:17-18` 中 `SES_SECRET_ID`/`SES_SECRET_KEY` 作为裸常量读取，但未在 `config.php` 中定义。

**已修复**：改为 `getenv("SES_SECRET_ID") ?: ""` 读取环境变量。

#### [x] 10. ticks=1 废弃

`task_daemon.php:41` 中 `declare(ticks = 1)` 在 PHP 8.x 中性能极差且触发废弃警告。

**已修复**：`declare(ticks = 1)` 替换为 `pcntl_async_signals(true)`。

#### [x] 11. 认证逻辑不一致

`delete_comment.php:6-8` 自己实现管理员检查，应使用 `check_admin()`。

**已修复**：改为 `include_once LIB_DIR . "auth.php"; if (!check_admin()) die(...)`。
